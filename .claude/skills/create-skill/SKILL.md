# create-skill

**Description**: 사용자의 요구사항을 분석하여 새로운 Claude Code 스킬 파일을 생성합니다. 스킬 이름, 목적, 실행 단계를 입력받아 `.claude/skills/<name>/SKILL.md`를 작성합니다.

## Trigger

사용자가 다음 중 하나를 언급할 때 이 스킬을 사용합니다:
- "스킬 만들어", "skill 생성", "새 스킬", `/create-skill`
- "~하는 스킬을 만들어줘"처럼 특정 동작을 수행하는 스킬 생성 요청

## Steps

### 단계 1: 요구사항 분석

사용자의 요청에서 다음 정보를 추출합니다. 명시되지 않은 항목은 합리적으로 추론합니다.

| 항목 | 추출 방법 |
|------|-----------|
| `name` | kebab-case 슬러그 (예: `mermaid-diagram`) |
| `description` | 한 문장 요약 — 무엇을 하는 스킬인지 |
| `trigger` | 어떤 말을 들었을 때 이 스킬을 실행할지 |
| `steps` | 구체적인 실행 단계 (도구 호출, 명령어 포함) |
| `output` | 완료 후 사용자에게 보고할 메시지 |

정보가 불충분하면 진행 전에 사용자에게 질문합니다.

### 단계 2: SKILL.md 초안 작성

아래 템플릿을 기반으로 SKILL.md 내용을 구성합니다.

```markdown
# <name>

**Description**: <description>

## Trigger

<언제 이 스킬을 실행하는지 — 키워드, 슬래시 커맨드, 상황>

## Steps

### 단계 1: <첫 번째 작업>

<구체적인 설명 — 사용할 도구, 실행할 명령어, 판단 기준>

### 단계 2: <두 번째 작업>

<...>

## 완료 보고

<완료 후 출력할 메시지>
```

작성 원칙:
- 각 단계는 Claude가 판단 없이 실행할 수 있을 만큼 구체적으로 작성
- 도구 호출이 필요한 경우 어떤 도구를 어떻게 쓸지 명시
- OS 분기, 오류 처리 등 예외 상황은 단계 내에 인라인으로 기술
- 주석이나 설명보다 실행 가능한 지시문 위주로 작성

### 단계 3: 파일 생성

```bash
mkdir -p .claude/skills/<name>
# Write 도구로 .claude/skills/<name>/SKILL.md 생성
```

생성 후 파일 경로를 확인합니다.

### 단계 4: 스킬 내용 요약 보고

생성된 스킬의 핵심 내용을 사용자에게 요약합니다:
- 스킬 이름 및 파일 경로
- 트리거 조건
- 실행 단계 개요

## 완료 보고

> `.claude/skills/<name>/SKILL.md` 스킬이 생성되었습니다.
> 다음에 "<trigger 키워드>"라고 말하면 이 스킬이 실행됩니다.
